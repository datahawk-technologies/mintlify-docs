## Overview

DataHawk exposes granular Amazon customer return records for both FBM and FBA. You can use this data to understand:

- Which products are returned most often
- When returns occur and how frequently
- Why customers return items
- Which returned items are damaged, sellable, or otherwise disposed

## Data tables

Returns data is available in the `SELLING_PARTNER` schema.

### FBM returns

- Table: `SELLING_PARTNER.RETURNS_DATA_SRC`
- Coverage: FBM or MFN orders
- Available since: June 2023
- Includes ASIN, SKU, quantities, return reason codes, reason descriptions, carrier information, and return-cost context

### FBA returns

- Table: `SELLING_PARTNER.FBA_CUSTOMER_RETURNS_SRC`
- Coverage: FBA orders
- Includes fulfillment center, disposition, customer comments when present, and detailed return reason codes

## FBA dispositions

| Disposition | Description |
| --- | --- |
| `SELLABLE` | Item can be resold and is returned to inventory |
| `DAMAGED` | Item was damaged by carrier or fulfillment center and Amazon reimburses the seller |
| `CUSTOMER DAMAGED` | Packaging was opened or item altered; not sellable but still held in inventory |
| `DEFECTIVE` | Product is faulty or non-functional |
| `EXPIRED` | Product is near or past expiration and may be removed or disposed |

These dispositions help you separate recoverable inventory from lost inventory and reimbursement cases.

| Unwanted item | `CR-UNWANTED_ITEM` | `UNWANTED_ITEM` |

## Join with order data

Use `finance.finance_orders` to enrich returns with order-level context such as fulfillment channel.

```sql
select
  o.fulfillment_channel,
  r.*
from SELLING_PARTNER.RETURNS_DATA_SRC r
left outer join finance.finance_orders o
  on o.order_id = r.order_id
```
